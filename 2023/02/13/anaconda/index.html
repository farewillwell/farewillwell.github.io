anaconda与Python包安装

软件与环境
发布日期:   2023-02-13

前言

笔者在整深度学习的时候，得到了一坨代码，这一坨代码里面引入了大量的库，有的Python本地自带，但是有的就没有，搞得笔者很难看，然后众所周知Python这个东西干啥主要是依靠包，本身自己的码没啥东西，所以如何解决这些包的下载成为了一个十分ex的东西

你可能会问，不对啊，你看你打开pycharm，直接出现报错就点击解决，下载下来就好了啊。

但是通过笔者从网上的查阅，和笔者亲自的实验，绝大多数这些要用到的包，都没能找到一个统一的渠道去完全的下载下来。

至于原因嘛，一个重要条件就是：前置环境

关于一些包，它的前置条件很坑人，非常的ex

比如笔者下载一个叫apex的包，然后这个东西需要去下一个NVIDIA的叫cuda的东西才行，关于这个cuda又有个学问了…笔者借此发现了一个叫迅雷的东西用来当浏览器下东西也蛮好用的

然后再比如一些包，需要Python的版本不能过高，不然它的包本身是要调用Python里的某些函数来下载的，结果发现随着Python版本的升级然后这个函数被取缔了绝望，当你看到这个报错的时候内心是绝望的。

下面分别细说

利用的东西

操作 功能
迅雷 下载大东西，比如下了个3.4G的cuda
anaconda 创建虚拟环境，这个下面细说
pycharm 基本的Python编程软件，能用jetbrain的还是用吧，比较好使确实
vs 据说这个下载cmake的时候有用，咱也搞不清个原理，不过反正以后写操作系统也要写cpp，不如 提前下载
pip Python自带的命令行，不过大多数情况不好用，我们需要充分利用镜像源
conda anaconda自带的命令行，个人感觉比pip好使，但是必须要在conda虚拟环境中使，笔者用这个攻克了好几个包的下载
各种镜像源 大清，豆瓣，阿里等等，十分的好用，能够很强的加速你的下载
GitHub 有些时候还是要直接从库里下载源码

anaconda与包安装

首先，理解anaconda是个什么东西

它可以于创造了多个Python的环境，可以在创造的环境里单独隔绝的运行Python程序（对不同版本的有奇效)

众所周知，在Python里，下载包，是十分必要的操作，包不行，什么都做不到，而下载包是一个很令人苦恼的过程，一些高级的包，并不能轻易下载出来，anaconda相当于给了我们一个下载包的好去处，相比于pip install，个人感觉下面的命令比较好用

conda install <name>

其中conda是anaconda的使用命令，install是安装，不解释.

而且这个东西相当于一个管理Python大量库的一个东西，通过它能很好的管理库（不过还是最稳妥的是上网单独搜库来下载）

笔者当时是为了解读ai的代码而去下载库

然后点击pycharm的相应功能发现下载失败（几乎这些库都下载失败了，绝望）

库能下载到哪里？

Python里面有一个叫做sys的东西，它代表了你包能用的路径。

在不同的解释器下，包能用的路径。

解释器是什么？

解释器，是你的电脑理解Python代码的方式，根据不同的解释器，你在代码里提到的东西会有不同的作用

不同库的下载方式

dlib

在anaconda里面使用

conda install dlib

发现失败了

其实是首先dlib是一个较多牵涉的库，需要下载很多前置的东西才行。

foundations

笔者下载换源啥的出了点问题，这里选择下载安装包
但是下载的时候输入这样的代码从豆瓣下载，出现了这个：NameError: name ‘reload’ is not defined
，Python版本的更替，以上代码已经被删除或者替换了，悲！也就是说这个版本的Python下不了这个。
在经典的报错，有个这个： 

reload(sys)
NameError: name 'reload' is not defined
[end of output]
  能在很多地方看见这个报错

（万万注意，还有一个包叫foundation，这俩不一样！！！）
然后生成了一个这个样的文件

foundations-0.1.0a0.dev1.tar.gz

该怎么使用这个包呢？
理论上这个包的使用方式：
1.用bandzip解压到一个文件夹里，然后从终端进去，然后点击setup.py
2.然后遇到报错：没reload这个东西，逼我改源码是吧
查资料发现

reload是一个函数，然后“在 Python2.x 版本中 reload() 是内置函数，在 Python2.x ~ Python3.3 版本移到 imp 包中(Python2.x 也可以导入 imp 包使用)，Python3.4 之后到版本移到了 importlib 包中。”

所以我们只需要把这个包导进来就行了

我们打开源码，找到load这一行

import sys
reload(sys)

就是这里的报错，说没这个函数，但是实际上这个函数是在importlib里的，用pip你也没法直接改源码，所以还是要从网站下下来改源码。

然后又出了一个bug（绝望）

AttributeError: module 'sys' has no attribute 'setdefaultencoding'

原因：Python 3 系统默认使用的就是utf-8编码，就不需要sys.setdefaultencoding(“utf-8”)这段代码，所以Python 3 的 sys 库里面已经没有 setdefaultencoding() 函数了！！删掉报错代码即可。
然后又报错

NameError: name 'reload' is not defined

原来是另一个函数里也有这个玩意，逼我大改md

改了之后又弹出来一个

sys.setdefaultencoding(DEFAULT_CODEC)
AttributeError: module 'sys' has no attribute 'setdefaultencoding'

还是已经没这个函数了，删掉删掉！
然后又报错。。

UnicodeDecodeError: 'gbk' codec can't decode byte 0x93 in position 4102: illegal multibyte sequence

行，然后发现需要这么改，找到那行代码：

with open(file) as f: 改成 with open(file, ‘r’, encoding=‘utf-8’) as f:

然后终于能跑了！但是出了更多的bug…但是至少下载下来了，报错还是依旧是reload没定义，我一怒之下全找一遍这个文件，看有没有其他的.
