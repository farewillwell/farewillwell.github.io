页表管理

emm

对于位于 kuseg 的虚拟地址，MOS 中采用两级页表结构对其进行地址转换

这句话，页表管理这个的操作是虚拟地址的如何找真实物理地址的页。

include/mmu.h 中提供了两个宏以快速获取偏移量，PDX(va) 可以获取虚拟地址 va 的 31-22
位，PTX(va) 可以获取虚拟地址 va 的 21-12 位。
访问虚拟地址时，先通过一级页表基地址和一级页表项的偏移量，找到对应的一级页表项，
得到对应的二级页表的物理页号，再根据二级页表项的偏移量找到所需的二级页表项，进而得到
该虚拟地址对应的物理页号。

流程：首先把地址移动到一级页表头，然后根据31-22的偏移找到对应的一级页表项，读出在哪个二级页表页，然后去这个页，从21-12的偏移找到对应的二级页表项，然后根据这个表项找到实际的页，根据最后几位的页偏移读入数据。

在 MOS 中，无论是一级页表还是二级页表，它们的结构都一样，只不过每个页表项记录的
物理页号含义有所不同。每个页表均由 1024 个页表项组成，每个页表项由 32 位组成，包括 20
位物理页号以及 12 位标志位。其中，标志位的定义规范和 EntryLo 寄存器的规范（详见 2.6.1）
相同（例如标志位 PTE_V、PTE_D 就分别对应表示 EntryLo 中的 V、D 标志位）。每个页表所占
的空间为 4KB，恰好为一个物理页面的大小

事实上，这两个的页表（而不是项）的本质都是页，所以创建的时候页往往用页来管理，页表项则是里面的数据。

表项的1024是这么来的，而且不是字节，是字的个数。

代码缩写对照
PDX - Page Directory indeX（⻚⽬录索引）
PTX - Page Table indeX（⻚表索引）
PDBase - Base address of Page Directory（⻚⽬录基地址）
PTBase - Base address of Page Table（⻚表基地址）
VA - Virtual Address（虚拟地址）
PA - Physical Address（物理地址）
Perm - Permission（权限）
VPN - Virtual Page Number（虚拟⻚号）
PFN - Physical Frame Number（物理⻚框号）

关键在于，谁是虚拟地址，谁是物理地址？

搞清楚的一个事实是：
1.无论是什么内置函数，操作的都是虚拟地址
2.几乎所有用到的东西都是虚拟地址，指针啥的也都是虚拟地址。

然后，究竟是怎么从页表项里面拿数据的？

页表本身，给定的都是虚拟地址，然后拿数据其实都是从虚拟地址拿的。

比如第一级页表，也就是页表目录，它的表头地址就是虚拟地址。（这个表头地址只有一个，放在内存里面）一个页表1024个页表项，所以它的第一部分是10位，第二个部分也是10位，然后一个物理页大小是2^12，所以最后一个部分12位。

然后拿来一个虚拟地址，要查页表（通过查页表才知道是在物理地址的哪里），首先拿过来分成三个部分。

然后第一个部分就是第几个页表目录项，直接在表头地址上加上就可以了（注意，第一个部分需要按照位的方式单独拿出来这几位，然后加也是加的这几位）

然后这里得到了一个虚拟地址，这个虚拟地址就是所在的第一级页表项的位置，它的内容，也就是第一级页表项，分为两段。

第一段是前20位，代表的是物理页号，也就是这个页表的下一级的数据对应的物理页号，然后后12位是权限。

注意，这个物理页号不是物理地址它确实跟物理页有关，然后能够提供下一级数据的物理序号。能够指出是到底是第几个物理页，也就是能给出物理地址所在的页的起始位置的物理地址（后面的需要再偏移）。然后注意，我们搞啥数据都是用的虚拟地址，然后经过把这个初始物理地址换算算出虚拟地址的位置，找到下一个页表的起始物理地址，根据二级页表的偏移找到二级页表项，然后根据这个项里面前20的位置找到物理页号，根据这个物理页号和实际的偏移找到物理地址。

然后在这个过程里需要这几个函数：

#define PDX(va) ((((u_long)(va)) >> 22) & 0x03FF)
#define PTX(va) ((((u_long)(va)) >> 12) & 0x03FF)
#define PTE_ADDR(pte) ((u_long)(pte) & ~0xFFF)

pdx是截取一级页表页号的

ptx是截取二级页表页号的

PTE_ADDR是将pte页表项（实际上前20位才是页表号数据）转化为物理地址（就是把物理页号后面补上全是0而已，因为物理页本身就是按照次序紧密排列的）

然后物理地址就拿到了，但是要计算的时候还是需要虚拟地址的。

Pde*一般指的是一级页表的项的地址

Pte*一般指的是二级页表的项的地址，一般可以用物理地址转虚拟地址得到。

时刻注意这些取地址得到的才是页表项。然后页表项本身存的经过PTEADDR就是物理地址。

	if (pte == NULL || (*pte & PTE_V) == 0) {
		return NULL;
	}
	如果没查到或者查到的无效，就是没有这个映射，返回
	/* Step 2: Get the corresponding Page struct. */
	/* Hint: Use function `pa2page`, defined in include/pmap.h . */
	pp = pa2page(*pte);
	if (ppte) {
		*ppte = pte;
	}
	查到了，那就让ppte的值变成查到的地址，让这个查到的映射页返回回去就行了。
	return pp;
}
/* Overview:
 *   Decrease the 'pp_ref' value of Page 'pp'.
 *   When there's no references (mapped virtual address) to this page, release it.
 */
void page_decref(struct Page *pp) {
	assert(pp->pp_ref > 0);

	/* If 'pp_ref' reaches to 0, free this page. */
	if (--pp->pp_ref == 0) {
		page_free(pp);
	}
}
// Overview:
//   Unmap the physical page at virtual address 'va'.
void page_remove(Pde *pgdir, u_int asid, u_long va) {
	Pte *pte;

	/* Step 1: Get the page table entry, and check if the page table entry is valid. */
	struct Page *pp = page_lookup(pgdir, va, &pte);
	if (pp == NULL) {
		return;
	}

	/* Step 2: Decrease reference count on 'pp'. */
	page_decref(pp);

	/* Step 3: Flush TLB. */
	*pte = 0;
	tlb_invalidate(asid, va);
	return;
}
