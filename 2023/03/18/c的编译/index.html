关于c语言编译

BUAA_OS
发布日期:   2023-03-18

gcc基础使用

假设将这个文件命名为 hello.c ，那么在终端中，你只要简单的敲下：

$ gcc hello.c

就会在当前目录下生成一个名为 a.out 的可执行文件。使用

$ ./a.out

对 C 而言，从源代码到可执行文件要经过四个步骤：预处理->编译->汇编->链接。这个完整的过程有时被直接称为“编译”或“构建”。这四个阶段都会调用独立的程序，对于 C 程序和 GNU 编译工具链而言，预处理阶段会以源文件作为输入，调用 cpp 生成预处理文件；编译阶段会以预处理后的文件作为输入，调用 cc1 生成汇编文件；汇编阶段会以汇编文件作为输入，调用 as 生成目标文件；最后以一个或多个目标文件作为输入，调用 ld 生成可执行文件。而 gcc 则是作为一个总的调度程序，调用这些真正执行处理的程序，将这些程序封装起来给用户提供了一个比较简明的操作界面，这样就降低了我们编译程序的难度。你可以在编译的时候用 –verbose 让 gcc 打印出详细的编译信息，看一看一个简单的 gcc hello.c 后隐藏了多少复杂的操作

gcc的参数

参数 意义
-E 只进行预处理,生成.i
-o 指定output 的file的名字（不写这个光写file默认把编译出来的放到这个文件夹里）
-S 生成汇编代码文件，生成.s
-c 生成机器码文件.out
-I 将指定目录的东西拿来链接，当然只要指定存在这个的文件夹就行了，不用指定文件，后面可以跟绝对地址，也可以./什么,也可以假如是上一层文件夹的就是../的，但是路径中间千万别空格,而且和参数之间留好空格！
预处理

重要要求：编出可执行文件gcc必须有main，所以不能单一一个文件编，否则会出undefined reference to `main
gcc不能编没有main的程序！，所以需要造一个main！
问题：重复使用该命令会重复编译吗？

在同一个文件夹之下，可以多次编译，只要名字不同即可，都会正常生成的.

可以使用 -E 选项让 gcc 只进行预处理操作：

$ gcc -E hello.c -o hello.i

预处理过程主要处理源代码文件中以 # 开始的预处理指令。主要处理规则是：

将所有的 #define 删除，并展开所有的宏定义。
处理所有的条件编译指令，如 #if ，#ifdef 等。
处理 #include 预处理指令，将被包含的文件内容插入到该预处理指令的位置。

预处理过程还执行了一些其它操作，当前可以先不了解，感兴趣的同学可以阅读预处理器。
编译

产生汇编代码。这一步是整个程序构建的核心部分，也是最复杂的部分之一，你在下学期的编译技术课上会深入学习这一部分的知识。如果想让 gcc 只从预处理文件生成汇编代码文件，可以使用 -S 参数，如下所示：

$ gcc -S hello.i -o hello.s

汇编

将汇编代码转换为机器代码。如果想让 gcc 只进行这一步操作，可以使用 -c 参数，如下所示：

$ gcc -c hello.s -o hello.o

此时生成的文件称作目标文件（object file）

链接

hello.c 中调用了 printf 这个函数用来输出字符串，但是你有没有想过这个 printf 函数的定义到底在哪里呢？你可能会想 printf 定义在 stdio.h 中，但实际上 stdio.h 中只有 printf 的声明，并没有这个函数的定义。链接就是要解决这个问题。简单而言，链接就是把多个目标文件的代码段放在一起、数据段放在一起，然后和一些写好的库函数（比如 scanf 和 printf ）放在一块共同形成可执行文件。

但是假如这个库在别的地方，会怎么样才能连接呢？比如在别的文件夹？

变量与存储

存储类别（storage class） 用来描述 C 语言中变量或者函数的一些特征。这些特征包括变量的存储期属性，作用域属性，以及链接属性。在 C 语言中，我们可以使用与存储类别有关的关键字（如 auto，extern，static）和变量声明的位置（如在函数内声明或者在函数外声明）来指定变量的存储类别。

为了说明变量的存储类别的内涵，我们需要先来了解一下变量的存储期，作用域和链接分别都是什么。

存储期

存储期（storage duration） 描述一个对象（object） 在内存中的生命周期。注意，这里出现了一个新的名词：对象（object）。C 语言中的对象与面向对象编程中的对象不是一个概念，一定不要混淆。在 C 语言中，对象就是连续的一片内存空间，具有起始地址与大小这两个属性。与对象对应的另一个概念是标识符（identifier）。标识符是一个符合 C 语言规定的变量命名规则的字符串，在程序中我们可以使用标识符访问修改对象

在 C 语言中，对象一共有四种存储期：静态存储期，自动存储期，动态分配存储期和线程存储期。

这里插入一段Cprimer里的内容

静态存储期：在函数中使用 static 关键字定义的变量，或者在函数外定义的变量(比如main函数外定义的在main执行的时候就是静态的)，它们所对应的对象具有静态存储期。如果对象具有静态存储期，那么它在程序执行期间一直存在，并且在程序执行过程中对象所占据的内存空间大小和对象的起始地址不会发生变化。如果不对该对象进行初始化，那么该对象的值会自动初始化为 0。由于这种对象在程序执行期间地址和大小不会发生变化，因此称为静态的。

自动存储期：在函数中不使用 static 关键字定义的变量所对应的对象具有自动存储期属性。程序执行到该变量声明的时候会创建变量对应的对象(比如函数里面声明，所以函数进行依次声明一次，每次的大抵都不一样)，在执行到该变量作用域结束时会释放掉该对象，随后该对象所占据的内存空间可做他用。如果在变量定义时不进行初始化，那么该变量的值不确定。因为对于对象的分配和释放并不需要程序员手动操作，因此称为自动的。也可以使用 auto 关键字在定义变量时显式指出该变量对应的对象具有自动存储期。注意：C 中的 auto 与 C++ 的 auto 是不一样的——前者表示自动存储期，后者表示自动类型推导

作用域

作用域（scope） 描述一个标识符在程序中可以被使用的区域。一个 C 变量具有的常见的作用域包括块作用域，函数作用域或者文件作用域。

块作用域（block scope）：块（block）是用一对花括号括起来的代码区域。定义在块中的变量具有块作用域，块作用域变量的可见范围是从定义处到包含该定义的块的结尾。此外，虽然函数的形式参数声明在函数体的花括号外，但是它们也具有块作用域，属于函数体这个块。

文件作用域（file scope）：如果变量定义在函数的外部，那么这个变量具有文件作用域。具有文件作用域的变量，从它的定义处到该定义所在的文件末尾均可见。由于这样的变量可以在多个函数中使用，因此也叫做全局变量。

链接

在 C 语言中，一个变量的链接（linkage） 属性描述了该变量是否可以在别的文件（即不是定义这个变量的文件）中被使用。变量具有 3 种链接属性：内部链接，外部链接或者无链接。

内部链接：内部链接变量只能在定义它的文件中使用。使用 static 关键字声明的全局变量是内部链接的。

外部链接：外部链接变量可以在所有的文件中使用。全局变量默认是外部链接的。

无链接：无链接其实就是这个变量没有链接属性的意思，在函数中定义的变量都是无链接的。注意无链接和内部链接的区别，无链接代表这个变量没有链接属性，没有办法将它变成外部链接或者内部链接的。而内部链接的变量去掉变量声明中的 static 关键字就变成了外部链接的，外部链接的变量在变量声明时加上 static 关键字就变成了内部链接的了

global参数可以指定是外部链接（物理全局）

使用命令 gcc foo.c hello.c 就可以将这两个 C 源文件编译成一个可执行文件了。如果想把编译和链接这两步拆开来做，可以使用这两条命令：

命令 gcc -c foo.c hello.c 对每个源文件依次进行编译操作，生成目标文件 foo.o 和hello.o。
命令 ld foo.o hello.o 链接两个目标文件，生成名为 a.out 的可执行文件。

extern int，这个东西代表是从外面的文件里面拿过来的，而不是自己定义生成的，和外面那个文件操作的变量都是一个东西

为什么只有使用 extern 关键字进行声明，我们才能访问别的文件中具有外部链接属性的变量呢？这里要介绍一下定义式声明（defining declaration） 和引用式声明（referencing declaration） 这两个概念。定义式声明会创建对象，而引用式声明不会创建对象，编译器会假设这个变量的定义在别的地方。对一个变量的定义式声明加上 extern 关键字后，这个声明就变成了引用式声明

变量的存储类别

在了解了 C 语言中变量的存储期，作用域与链接这三个概念后，我们终于可以将这三个概念组合起来，来完整的描述一个变量的存储类别。注意：存储类别并不是变量所具有的一个独立的属性，而可以认为是变量所具有的存储期，作用域与链接这三个属性的统称。

我们可以使用存储类别说明符与变量定义的位置来指定变量的存储类别：

存储类别说明符 变量声明位置 存储期 链接 备注
auto 函数内 自动存储期 无链接 auto 关键字可以省略
static 函数内 静态存储期 无链接
static 函数外 静态存储期 内部链接
无 函数外 静态存储期 外部链接 若要在别的文件中使用这种变量需要使用 extern 关键字进行引用式声明
extern 函数内 静态存储期 引用的变量需要具有外部链接 引用式声明，不会创建对象
extern 函数外 静态存储期 引用的变量需要具有外部链接 引用式声明，不会创建对象

总结关于static这个关键字，加上之后它的声明周期都是变长的，都是一直存在的了，函数内的也始终是一个东西，但是可连接性变差了

常量指针和指针常量

const int *p：（理解，int p指的是p是int的，这个值不可变）
const 和 int 可以互换位置，二者是等价的。我们以放在最前面时进行描述。

当把 const 放最前面的时候，它修饰的就是 *p，那么 p 就不可变。p 表示的是指针变量 p 所指向的内存单元里面的内容，此时这个内容不可变。其他的都可变，如 p 中存放的是指向的内存单元的地址，这个地址可变，即 p 的指向可变。但指向谁，谁的内容就不可变。

这种用法常见于定义函数的形参。前面学习 printf 和 scanf，以及后面将要学习的很多函数，它们的原型中很多参数都是用 const 修饰的，这样做的好处是安全！我们通过参数传递数据时，就把数据暴露了。而大多数情况下只是想使用传过来的数据，并不想改变它的值，但往往由于编程人员个人水平的原因会不小心改变它的值。这时我们在形参中用 const 把传过来的数据定义成只读的，这样就更安全了。这也是 const 最有用之处。

int*const p=&a；
此时 const 修饰的是 p，所以 p 中存放的内存单元的地址不可变，而内存单元中的内容可变。即 p 的指向不可变，p 所指向的内存单元的内容可变。

梳理：

C代码的经过：预处理->编译->汇编->链接

这几个阶段是这样的：

预处理

a.c经过预处理->a.i

操作：展开#define和include

单独执行参数： -E

编译

a.i经过处理变成a.s，这个.s叫做汇编文件，是用mips之类的东西些的

操作：编译

单独执行参数： -S

汇编

汇编和编译不是一个东西，汇编生成的是机器码，为a.o，是目标文件
单独执行：-c

连接：

生成可执行文件，是将各种 库 的东西连接起来，各种声明啥的，这时候才需要其他库其他文件啥的，前面的全都是自己的。
